# Sprint 1 — Foundation & Architecture

## Goal
Stand up the **pnpm monorepo** at `D:\trimly_final` and bring all four apps to a **verified, runnable** state with shared tooling, environment management, Docker dev-infra, and documentation. No business features yet — this sprint is purely the architectural foundation that every later sprint builds on. After this, you can `pnpm dev` the backend + admin panel and `flutter run` both apps.

This sprint covers spec-line **"Project setup & architecture"** and lays the rails for **"Multi-tenant foundation"** + **"Database & Prisma schema"** (which begin in Sprint 2).

**Why:** one source of truth for the DB schema and shared types, one CI pipeline, dependency sharing, and the apps stay independently deployable.

## Cross-cutting (root)

- **`.env.example`** — fully documented keys (no real secrets), grouped: `APP_*`, `DATABASE_*`, `REDIS_*`, `JWT_*`, `OTP_*`, `RAZORPAY_*`, `S3_*`/`R2_*`, `FIREBASE_*`, `SMTP_*`, `TWILIO_*`/`WHATSAPP_*`, `GOOGLE_*`, `APPLE_*`, `SOCKET_*`, `POSTHOG_*`. App loads via `@nestjs/config` (backend) and `envied`/`process.env` (flutter).
- **`docker-compose.yml`**: `postgres:16-alpine` (volume, healthcheck), `redis:7-alpine`, `redisinsight` (optional), ports on env. `docker compose up -d` is the one-command infra start.
- **CI** `.github/workflows/ci.yml`: matrix `lint` → `typecheck` → `test` → `build` for the JS workspace; `flutter analyze` + `flutter test` for both Flutter apps. Runs on `ubuntu-latest`, caches pnpm + pub.
- **Turborepo** `turbo.json`: `build`, `dev`, `lint`, `test`, `typecheck`, `db:migrate` with correct `dependsOn` and `outputs` (`.next/`, `dist/`).
- **`.gitignore`** merged; **`.gitattributes`** forces LF.
- **`README.md`** root: architecture overview, prerequisites, quickstart (`docker compose up -d`, `pnpm install`, `pnpm db:migrate`, `pnpm dev` / `flutter run`), env guide, sprint roadmap status.
- **`AGENTS.md`** root: supersedes the scaffold's note — documents monorepo conventions, the Next-16-read-the-local-docs rule, commit/branch conventions, where DB migrations live, and how to run tests.
- **Git**: init at root (not currently a repo), `.gitignore` in place, initial commit of the Sprint-1 foundation. (Per your earlier answers, no remote push unless you ask.)
